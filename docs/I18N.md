# Internationalization (i18n)

Trenerka frontend uses [i18next](https://www.i18next.com/) + [react-i18next](https://react.i18next.com/) with browser language detection.

## Supported languages

| Code | Language | Notes |
|------|----------|-------|
| `ru` | Russian | Default fallback |
| `en` | English | Source for many secondary locales |
| `de` | German | |
| `pt` | Portuguese | |
| `ja` | Japanese | |
| `it` | Italian | |
| `es` | Spanish | |
| `fr` | French | |
| `ar` | Arabic | RTL — `dir="rtl"` on `<html>` when selected |
| `zh-CN` | Chinese (Simplified) | |

User choice is stored in `localStorage` under key **`trenerka_lang`**.

## Folder structure

```
frontend/src/i18n/index.ts          # i18n init, supported languages, RTL helper
frontend/src/locales/{lang}/        # Per-locale JSON namespaces
  common.json
  landing.json
  auth.json
  trainer.json
  client.json
  admin.json
frontend/src/components/shared/LanguageSwitcher.tsx
```

## Namespaces

| Namespace | Use for |
|-----------|---------|
| `common` | Buttons (Save, Cancel, Login), site nav/footer, Skills label |
| `landing` | Marketing / landing page |
| `auth` | Login, register, validation messages |
| `trainer` | Trainer layout nav, exercises, files, settings |
| `client` | Client shell, profile |
| `admin` | Admin UI (minimal) |

## Adding a new string

1. Add the key to **`frontend/src/locales/ru/{namespace}.json`** (primary copy).
2. Add the same key to **`en`** and other locale files (or run `node frontend/scripts/generate-locales.mjs` after updating the script’s `T` object).
3. In a component:

```tsx
import { useTranslation } from 'react-i18next'

function MyComponent() {
  const { t } = useTranslation('trainer') // or ['trainer', 'common']
  return <button>{t('exercisesPage.title')}</button>
}
```

4. Interpolation: `t('exercisesPage.catalog', { count: 12 })` with JSON `"catalog": "Catalog of {{count}} exercises"`.
5. Cross-namespace: `t('common:actions.save')` when `useTranslation` includes `common`.

## Language switcher

`LanguageSwitcher` is mounted in:

- Site header (landing / public pages)
- Trainer layout header
- Trainer settings page

Dropdown shows **native** language names (Deutsch, Português, 日本語, …).

## RTL (Arabic)

`applyDocumentDirection()` in `src/i18n/index.ts` sets `document.documentElement.dir` to `rtl` for `ar`, `ltr` otherwise. Called on init and on `languageChanged`.

## Regenerating locale files

After editing `frontend/scripts/generate-locales.mjs`:

```bash
cd frontend && node scripts/generate-locales.mjs
```

## Build

```bash
cd frontend && npm run build
```

`tsconfig.app.json` has `resolveJsonModule: true` for JSON locale imports.
